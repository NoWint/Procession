# Procession 视觉特效系统设计

> **文档身份**：本文件定义 Procession 的视觉特效语言。它不是“让画面更炫酷”，而是“让数字世界看起来像真实存在”。所有光照、材质、粒子、后处理、环境效果的实现都必须回到这里验证：它是否增强了用户对“这是一个真实数字文明”的信念。

## 1. 设计宣言：视觉不是装饰，是世界存在的证据

Procession 的视觉效果必须服务于一个核心目标：

**让用户相信，他正在观察一个真实存在的计算宇宙。**

因此：

- 光必须像数据产生的自然现象。
- 材质必须像数字生命的皮肤。
- 粒子必须像空气中的尘埃与能量。
- 天气必须像系统状态在环境中的投射。
- 动画必须像世界在呼吸。

视觉特效不是游戏特效，不是电影爆炸，不是霓虹灯招牌。它是一种克制的、高级的、科学的、有生命的视觉语言。

## 2. 核心设计原则

### 2.1 克制的高级感

- 以深色空间为画布，以少量电光青、冷蓝为能量色。
- 禁止使用高饱和彩虹色、廉价霓虹、过度 Bloom。
- 每一个光效都应有来源：CPU 活动、网络流量、进程生命周期。

### 2.2 数据驱动的自然现象

- CPU 高负载 → 热浪、光强增强、能量流加速。
- 网络高峰 → 数据雨、光缆密度增加、粒子流动加快。
- 内存压力 → 空间膨胀感、建筑体积增大、环境轻微扭曲。
- 系统稳定 → 宁静的数字晨光、低频呼吸光。
- 异常 / 崩溃 → 局部红色扰动、结构损伤、空气噪波。

### 2.3 生命感动画

- 禁止线性淡入淡出、硬切、瞬间出现。
- 所有效果必须经历：酝酿 → 发生 → 持续 → 衰减 → 消散。
- 动画节奏受系统状态调制：高负载时世界更“紧张”，低负载时更“宁静”。

### 2.4 空间连续性

- 视觉效果必须存在于三维空间中，而不是贴在屏幕上。
- HUD 不应有漂浮的特效，世界的变化本身即为信息。
- 后处理（景深、运动模糊）必须强化空间深度，而不是掩盖细节。

## 3. 光照系统

### 3.1 整体方向

融合：

- Apple Vision Pro 空间计算的柔和体积光。
- Blender 电影级渲染的精确光比。
- Unreal Engine 环境效果的氛围感。
- 科学神经可视化的高级克制。

### 3.2 光源类型

| 光源 | 来源 | 视觉特征 | 数据驱动 |
|------|------|----------|----------|
| 核心光源 | 城市中心 / Kernel Core | 稳定、低频、冷白 | CPU 基线活动 |
| 建筑自发光 | 建筑外壳与内部 | 柔和、有呼吸感 | CPU + 内存 |
| 能量流发光 | 光缆与管道 | 方向性、脉动 | 网络 + IO |
| 环境散射 | 粒子与体积雾 | 微弱、弥漫 | 整体系统负载 |
| 事件光源 | 异常 / 进程诞生消亡 | 短暂、聚焦 | 生命周期事件 |

### 3.3 体积光（Volumetric Light）

- 在城市中央与高能建筑顶部使用细微的体积光柱。
- 体积光必须非常克制，只在高 CPU 或特殊事件时明显。
- 实现方案：Three.js 自定义 Shader 或 WebGPU 后处理体积光。

### 3.4 动态全局光照感

- 不使用真实 GI（性能过高），而是通过环境光与建筑自发光模拟 GI 感觉。
- 高负载区域整体变亮，低负载区域沉入黑暗。
- 建筑之间的光会相互染色（使用 Additive 混合与雾效）。

### 3.5 Bloom 规范

- 使用 `UnrealBloomPass` 但参数必须克制：
  - `strength: 0.2–0.4`
  - `radius: 0.3–0.6`
  - `threshold: 0.6–0.85`
- Bloom 只应用于真正的强光源，避免整个世界发糊。

## 4. 数据流效果

### 4.1 数据流的本质

数据流是计算机内部的信息流动，在城市中表现为：

- 建筑之间的光缆。
- 进程内部的数据管线。
- 网络连接的数据包粒子。
- 内存读写形成的光河。

### 4.2 光缆（Network Cables）

- **几何**：CatmullRomCurve3 + TubeGeometry，或使用 GPU 线条渲染。
- **动画**：光粒子沿曲线运动，方向 = 数据流向，速度 = 带宽。
- **密度**：由连接活跃度决定；高网络负载时粒子密度增加、速度加快。
- **颜色**：按协议映射（HTTP 青、SSH 绿、Database 紫、系统服务蓝等），但整体饱和度需降低以符合克制原则。

### 4.3 能量管道（Process Pipes）

- 连接父子进程与同家族进程的能源管线。
- 管道内流动着代表 CPU / 内存 / IO 的能量光。
- 管道粗细代表关系强度，亮度代表活动水平。

### 4.4 数据雨（Data Rain）

- 网络高峰时，天空中出现细微的垂直光丝落下，像数字雨。
- 不是 Matrix 式绿雨，而是半透明的青蓝色光丝，密度低、速度受网络吞吐调制。
- 只应在 Network Lens 或高网络负载时明显。

### 4.5 内部数据流（建筑内）

- 进入 Architecture Layer 后，建筑内部可见：
  - 内存分配形成的光河在楼层间流动。
  - 线程活动像萤火虫一样移动。
  - 网络 IO 像血管一样脉动。

## 5. 建筑材质

### 5.1 材质类型

建筑不是普通模型，而是数字生命的皮肤。

| 材质 | 视觉 | 用途 |
|------|------|------|
| 透明晶体 | 半透明、内部结构可见、折射光 | 浏览器 / 信息类应用 |
| 磨砂陶瓷 | 哑光、温润、低反射 | 系统服务 / 内核建筑 |
| 液态金属 | 细腻渐变、边缘流动发光 | 选中状态 / 高亮轮廓 |
| 神经网络表面 | 表面有细微脉冲纹路 | 高复杂度进程 / AI 相关进程 |
| 损伤材质 | 暗斑、裂纹、不稳定闪烁 | 崩溃过或异常进程 |

### 5.2 材质与进程类型的映射

| 进程类型 | 主材质 | 特征 |
|----------|--------|------|
| Kernel | 磨砂陶瓷 + 冷白金属 | 厚重、稳定、低频脉动 |
| 浏览器 | 透明晶体 + 信息玻璃 | 通透、扩张、连接密集 |
| 代码编辑器 | 工业钛灰 + 液态金属边缘 | 精密、机械、管线外露 |
| 游戏 | 有机-几何混合 + 动态纹理 | 活跃、变化、能量充沛 |
| 媒体工具 | 流动形态 + 暖色能量 | 柔软、创作感 |
| 系统服务 | 规则重复陶瓷 | 低调、稳定 |

### 5.3 建筑内部可视化

- 建筑外壳半透明化时，内部可见：
  - 能源核心：CPU 活动的光球。
  - 资源管线：内存流动的管道。
  - 线程人群：代表线程的小光点移动。
  - 网络连接：从端口伸出的光线。

### 5.4 程序化表面纹理

- 使用 GLSL Noise + SDF 生成动态表面纹理。
- 纹理变化由 CPU / 内存活动驱动。
- 避免使用外部贴图，全部程序化生成以保证独特性。

## 6. 粒子系统

### 6.1 粒子类型

| 粒子 | 含义 | 行为 |
|------|------|------|
| 数据星尘 | 背景活动 / 空闲资源 | 缓慢漂浮、无方向 |
| 能量粒子 | CPU / IO 活动 | 沿管道/光缆运动 |
| 诞生粒子 | 新进程生成 | 从地面向目标聚合 |
| 消亡粒子 | 进程关闭 | 从建筑解体、回归地面 |
| 网络包 | 网络数据 | 沿光缆高速运动 |
| 线程光点 | 线程活动 | 在建筑内部移动 |
| 天气粒子 | 环境天气 | 受系统状态调制 |

### 6.2 GPU 粒子系统

- 所有粒子使用 GPU 驱动（Three.js Points + ShaderMaterial，或 WebGPU Compute Shader）。
- 支持 5000–20000 粒子 @ 60fps。
- 每个粒子具有：位置、速度、生命周期、尺寸、颜色、透明度。

### 6.3 粒子行为逻辑

```glsl
// 伪代码
particle.position += particle.velocity * deltaTime;
particle.life -= deltaTime;
particle.opacity = smoothstep(0.0, 0.2, particle.life) * smoothstep(1.0, 0.8, particle.life);
particle.size = baseSize * (1.0 + sin(time * frequency + phase) * 0.3);
```

### 6.4 粒子与系统状态联动

- CPU 低：粒子漂浮缓慢，密度低。
- CPU 高：粒子速度加快，密度增加，轨迹更明显。
- 网络高：网络包粒子数量激增，形成数据雨。
- 系统稳定：粒子运动呈现规律的呼吸节奏。

## 7. 环境天气系统

### 7.1 天气是系统状态的自然现象

将计算机状态转化为数字世界的天气：

| 系统状态 | 天气现象 | 视觉表现 |
|----------|----------|----------|
| CPU 高负载 | 热浪风暴 | 空气扭曲、光柱颤动、远处景物轻微模糊 |
| GPU 长时间运行 | 能源极光 | 天空中出现青绿色光带缓慢流动 |
| 网络高峰 | 数据雨 | 垂直光丝落下，密度与速度随网络变化 |
| 内存压力 | 空间膨胀 | 建筑体积增大、环境轻微扭曲、粒子扩散 |
| 系统长期稳定 | 数字晨光 | 柔和的环境光、低频呼吸、粒子缓慢漂移 |
| 异常 / 崩溃 | 扰动风暴 | 局部红色噪波、结构闪烁、空气不稳定 |

### 7.2 热浪效果

- 使用 GLSL 扭曲 Shader 在建筑周围实现微弱热浪。
- 热浪强度与 CPU 成正比。
- 必须非常克制，只在高负载区域明显。

### 7.3 能源极光

- GPU 长时间运行时，城市上空出现缓慢的极光光带。
- 颜色：电光青到冷蓝的渐变。
- 运动：缓慢波动，像真实极光一样有层次感。

### 7.4 数据雨

- 网络高峰时，天空降落半透明光丝。
- 颜色：冷蓝 / 电光青。
- 密度：由网络吞吐决定。
- 必须避免 Matrix 式绿色瀑布感。

### 7.5 数字晨光

- 系统稳定运行时，整个城市笼罩在柔和的晨光中。
- 光线方向固定但强度缓慢呼吸。
- 建筑自发光降低，环境散射增强。

### 7.6 扰动风暴

- 异常或崩溃时，局部区域出现红色/琥珀色扰动。
- 空气有噪波扭曲，建筑边缘不稳定闪烁。
- 必须避免恐怖感，而是呈现“系统正在自我修复”的科技感。

## 8. Shader 方向

### 8.1 核心技术栈

- **GLSL Shader**：用于材质、粒子、后处理。
- **SDF（Signed Distance Field）渲染**：用于程序化建筑形态、内部结构。
- **Procedural Material**：使用 Noise + FBM 生成动态纹理。
- **Ray Marching**：用于体积光、云雾、特殊建筑内部效果。
- **Compute Shader**：用于大规模粒子系统与数据流模拟。

### 8.2 关键 Shader

| Shader | 用途 | 输入 |
|--------|------|------|
| 建筑表面 Shader | 动态纹理、能量脉动 | CPU、内存、类型、时间 |
| 数据流 Shader | 光缆内粒子运动 | 网络数据、时间 |
| 热浪 Shader | 空气扭曲 | CPU、温度、时间 |
| 极光 Shader | 天空光带 | GPU、时间 |
| 体积光 Shader | 光柱、光束 | 光源位置、强度 |
| 后处理 Shader | Bloom、景深、颗粒 | 渲染结果、深度、运动向量 |

### 8.3 Noise 函数选择

- 建筑纹理：Simplex Noise + FBM。
- 热浪扭曲：Perlin Noise 时间扭曲。
- 极光：Layered Sine + Perlin。
- 天气：Voronoi + FBM。

## 9. 后处理

### 9.1 后处理栈

```
Render Scene
    ↓
UnrealBloomPass（克制 Bloom）
    ↓
Depth of Field（轻微景深）
    ↓
Motion Blur（极轻微运动模糊）
    ↓
Film Grain（微弱胶片颗粒）
    ↓
Chromatic Aberration（极低色散）
    ↓
Vignette（轻微暗角）
    ↓
Output
```

### 9.2 各项参数

#### Bloom

- `strength: 0.25`
- `radius: 0.45`
- `threshold: 0.75`

#### Depth of Field

- 只在 Focus 模式与 Architecture Layer 使用。
- 焦距对准选中建筑，背景适度虚化。
- Bokeh 半径保持极小，避免过度梦幻感。

#### Motion Blur

- 只在摄像机快速运动时轻微启用。
- 强度极低，主要用于强化速度感。

#### Film Grain

- 强度 3–5%，增加电影质感。
- 颗粒应细腻，不干扰数据阅读。

#### Chromatic Aberration

- 强度 0.5–1%，只在画面边缘。
- 用于模拟高端光学仪器的轻微色差。

#### Vignette

- 轻微暗角，引导视线向中心集中。
- 颜色与当前环境色温一致。

### 9.3 后处理性能预算

- 后处理总开销 < 3ms / 帧。
- 在性能模式下可降低 Bloom 分辨率、关闭景深与运动模糊。

## 10. 声音联动（预留）

视觉与系统状态同步，声音作为可选增强层。

### 10.1 声音设计原则

- 极低音量，默认几乎不可闻。
- 声音不是音乐，而是环境音与仪器音。
- 用户可随时完全关闭。

### 10.2 声音映射

| 视觉事件 | 声音 |
|----------|------|
| UI 出现/消失 | 极低频空间“咔哒”或“呼吸”声 |
| 高负载时 HUD | 轻微电流嗡鸣 |
| 进入建筑 | 短促的能量聚焦音 |
| 进程诞生 | 微小的能量聚合声 |
| 进程消亡 | 低沉的结构解体声 |
| 网络高峰 | 细微的数据流水声 |
| 系统稳定 | 宁静的环境底噪 |

## 11. 性能与扩展

### 11.1 视觉特效预算

| 效果 | 预算 |
|------|------|
| 粒子系统 | 5000–20000 GPU 粒子 |
| 光缆 | 最多 200 条，使用 GPU 线条 |
| 体积光 | 最多 5–10 束，远处降级 |
| 后处理 | < 3ms / 帧 |
| Shader 复杂度 | 在 Intel 核显上保持 30fps |

### 11.2 性能模式

- **画质模式**：完整后处理、全粒子、体积光、动态阴影。
- **平衡模式**：中等 Bloom、粒子减半、无体积光。
- **性能模式**：仅基础 Bloom、最小粒子、无景深/运动模糊。

### 11.3 关键技术

- **GPU Instancing**：建筑、粒子、光缆统一实例化。
- **Compute Shader**：粒子系统与数据流使用 WebGPU。
- **LOD**：远距离特效降级或关闭。
- **Frustum Culling**：只渲染视锥内对象。
- **Texture Atlas**：减少材质绑定切换。

- [ ] 后处理总开销 < 3ms / 帧。

## 13. 禁止清单

- 禁止强烈霓虹与彩虹色泛滥。
- 禁止过度 Bloom 导致画面发糊。
- 禁止游戏 HUD 式特效堆叠。
- 禁止廉价赛博朋克风格。
- 禁止无数据来源的随机闪烁与爆炸。
- 禁止瞬间出现/消失的视觉变化。
- 禁止高饱和度糖果色。
- 禁止让 UI 元素本身产生装饰性粒子。
