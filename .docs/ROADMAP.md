# Procession 路线图

## 阶段总览

```
Phase 1 ──── Phase 2 ──── Phase 3 ──── Phase 4 ──── Phase 5
脚手架       城市初现       网络光缆       产品打磨       无限可能
(1-2周)      (2-3周)       (1-2周)       (2-3周)       (远期)
```

---

## Phase 1 — 脚手架 · "让城市有地基建"

**目标：** 打通 Tauri 全链路，能在屏幕上看到代表进程的立方体。

| 任务 | 负责 | 说明 |
|------|------|------|
| Tauri 2.x 项目初始化 | 严梓峻 | React + TypeScript + R3F 模板 |
| Rust SystemEngine 骨架 | 严梓峻 | PlatformAdapter trait + Windows 基础实现 |
| sysinfo + CPU/内存采集 | 严梓峻 | 实现 `get_cpu()` `get_memory()` `get_processes()` |
| DataBridge + IPC 推送 | 严梓峻 | 1Hz 推送 SystemSnapshot 到前端 |
| useSystemData hook | 夏天 | 接收 Tauri event，驱动 React state |
| R3F 场景骨架 | 夏天 | Camera + Lights + OrbitControls + 空场景 |
| 测试立方体 | 夏天 | 用测试数据渲染 3-5 个按 CPU 变高的方块 |

**验收标准：** 启动 app → 能看到方块在没有坐标轴的空间中浮动，方块高度随 CPU 变化。

---

## Phase 2 — 城市初现 · "让建筑群有逻辑"

**目标：** 真实数据驱动城市布局，InstanceMesh 高效渲染几百个建筑。

| 任务 | 负责 | 说明 |
|------|------|------|
| 进程树布局算法 | 严梓峻 + 夏天 | 根进程居中，子进程沿层级向外辐射 |
| InstancedMesh 建筑 | 夏天 | 替换测试立方体，映射真实数据 |
| 建筑颜色系统 | 夏天 | 系统/用户/活跃三色系 + 辉光 |
| 地面网格系统 | 夏天 | 半透明发光网格 + 微动态 |
| 天空/氛围系统 | 夏天 | 粒子背景 + UnrealBloomPass 辉光 |
| 进程详情浮窗 | 夏天 | 点击建筑 → 显示 PID/CPU/内存/状态 |
| 相机交互 | 夏天 | OrbitControls + 双击飞向 |
| 颜色参数定义 | 夏天 | 可调整的配色方案 JSON |

**验收标准：** 打开 Chrome / 运行 Node 等操作，城市中对应建筑"长高发光"；点击建筑看详情。

---

## Phase 3 — 网络光缆 · "让城市活起来"

**目标：** 建筑之间有流动的光缆，城市真正"活"了。

| 任务 | 负责 | 说明 |
|------|------|------|
| 网络连接采集 | 严梓峻 | `get_network()` 含连接列表（Windows） |
| 连接 → 光缆映射 | 夏天 | 对端 IP → 城市相对位置映射 |
| LineGeometry 渲染 | 夏天 | 建筑间发光线条 |
| 粒子流系统 | 夏天 | 沿光缆流动的光点动画 |
| 光缆颜色 (协议映射) | 夏天 | TCP=蓝, UDP=绿, HTTP=青... |
| 建筑顶部光晕脉动 | 夏天 | Running 进程顶端闪烁效果 |

**验收标准：** 打开浏览器下载文件 → 看到对应建筑间亮起光缆，粒子沿缆线流动。

## Phase 4 — 产品打磨 · "让 Procession 值得骄傲"

**目标：** 补齐 macOS、实用模式、性能优化、发布。

| 任务 | 负责 | 说明 |
|------|------|------|
| macOS PlatformAdapter | 严梓峻 | sysinfo + IOKit 适配 |
| GPU / 温度检测 | 严梓峻 | 可选传感器数据 |
| 空格键实用模式 | 夏天 | 建筑标注 + HUD 仪表盘叠加 |
| HUD StatsPanel | 夏天 | 实时 CPU/内存/网络概览 |
| 性能优化 | 双方 | 进程上限 1000+、LOD、帧率稳定 60fps |
| Tauri 打包配置 | 严梓峻 | Windows MSI + macOS DMG |
| 颜色主题系统 | 夏天 | 用户可选的配色方案 |
| 基础错误处理 | 双方 | 数据丢失降级、空状态、平台不兼容提示 |
| README + 宣传视频 | 双方 | GitHub 仓库准备开源 |

**验收标准：** Windows + macOS 都能跑，有实用模式，可打包安装。

## Phase 5 — 无限可能 · "远景规划"

这些是核心设计之外的增值方向，按兴趣和反馈陆续开展：

- **进程关系图谱** — 显示进程间 fork 和通信关系
- **端口可视化** — 监听端口的"港口"效果
- **时光回放** — 拖动时间轴，看过去几分钟的城市变化
- **文件系统热点** — 最近读写频繁的目录作为"热区"高亮
- **自定义主题** — 用户自定义建筑/City 配色、光缆样式
- **屏保模式** — 全屏运行，屏幕保护级别
- **插件系统** — 第三方数据源接入
- **社区分享** — 截图/动图分享你的 Procession 城市

## 技术债务与风险管理

| 风险 | 缓解措施 |
|------|---------|
| macOS 开发环境不足 | 前期用 cross-compilation，后期借 Mac 真机测试 |
| 系统 API 权限限制 | Tauri 自动提权脚本 + 用户引导，不强制提权 |
| 进程数过多 (1000+) | InstancedMesh + LOD + 进程数上限可配置 |
| 3D 性能不足在低配机器 | 允许关闭 bloom、降低建筑细节、限制 FPS |
| Windows/macOS API 差异大 | PlatformAdapter 隔离，Mock 测试 |
