## 二、核心定位

Procession 将电脑的运行状态实时映射为一座动态的 3D 城市景观。它是一个全新的品类——**不是更好看的任务管理器，而是让系统监控成为桌面艺术**。

**设计哲学：**
- 默认是桌面艺术品，好看是第一要务
- 交互后是实用工具，信息密度足够
- 不依赖云，一切本地处理
- 数据不撒谎——城市状态严格对应真实系统

**目标用户：** 开发者、技术爱好者、桌面美化爱好者

**目标平台（首发）：** Windows 10/11，macOS 紧随，Linux 暂不考虑

## 三、视觉隐喻（一句话说清）

> 每个进程是一座建筑，CPU 是建筑高度，内存是底面积，进程树决定街区布局，网络连接是建筑间的发光光缆。整个系统像一座活着的赛博城市。

## 四、阶段路线图

总工期预估：**6-10 周** 到 v0.1 Alpha 发布。分四个核心阶段加一个远景。

### Phase 1 — 脚手架（1-2 周）
**目标：** 打通全链路，能看到代表进程的方块在屏幕上随 CPU 变化。

**关键交付：**
- Tauri 2.x 项目初始化，前后端打通
- Rust 后端采集 CPU、内存、进程列表
- IPC 通道建立，1Hz 数据推送到前端
- R3F 场景骨架 + 测试立方体渲染

**验收标准：** 启动 app → 看到方块在 3D 空间中随 CPU 变化高度。

### Phase 2 — 城市初现（2-3 周）
**目标：** 真实数据驱动城市，高效渲染数百座建筑。

**关键交付：**
- 进程树辐射布局算法
- InstancedMesh 建筑群（单次 draw call 渲染所有建筑）
- 建筑颜色系统（系统蓝/用户橙/活跃白金）
- 发光网格地面 + 星空粒子氛围
- 建筑详情浮窗 + OrbitControls 相机交互

**验收标准：** 打开 Chrome → 城市中对应建筑长高发光；点击建筑查看详情。

### Phase 3 — 网络光缆（1-2 周）
**目标：** 建筑间有流动光缆，城市真正"活"起来。

**关键交付：**
- 网络连接采集（TCP 连接表）
- 光缆渲染（连接对端 IP → 建筑间发光曲线）
- 粒子流动画（光点沿缆线流动）
- 按协议区分光缆颜色（HTTP 青、SSH 绿、DB 紫等）

**验收标准：** 打开浏览器下载文件 → 看到对应建筑间亮起光缆，粒子流动。

### Phase 4 — 打磨发布（2-3 周）
**目标：** 跨平台可用、实用模式、性能达标、可打包安装。

**关键交付：**
- macOS PlatformAdapter 适配
- 空格键切换实用模式（建筑标签 + HUD 仪表盘）
- 性能优化（FPS 稳定 60，支持 1000+ 进程）
- 配色主题系统 + 设置面板
- Windows MSI + macOS DMG 安装包

**验收标准：** Win + Mac 都能跑，有实用模式，可打包安装。

### Phase 5 — 远景（长期）
增值方向，按兴趣和反馈陆续开展：
- 进程关系图谱
- 时光回放（拖动时间轴看城市变化）
- 自定义主题
- 屏保模式
- 插件系统

## 七、核心风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| macOS 开发环境不足 | 前期 cross-compilation，后期借 Mac 真机测试 |
| 系统 API 权限限制 | Tauri 自动提权 + 用户引导，不强制提权 |
| 进程数过多（1000+） | InstancedMesh + LOD + 用户可设置上限 |
| 低配机器 3D 性能差 | 自动关闭 Bloom、降低建筑细节、限制 FPS |
| Win/Mac API 差异大 | PlatformAdapter 隔离系统调用，Mock 测试 |

## 八、竞品区分

不要和传统的系统监控工具（任务管理器、htop、Rainmeter）比较。Procession 的定位不是"功能更全的监控器"，而是**让监控成为艺术**的新品类。竞品在信息密度上更优，Procession 在**情感感知**上不可替代——用户瞟一眼城市的"天气"就能直觉感知系统状态，而不需要读数字。
