# Procession 前端设计图对齐方案 v2

> **文档身份**：在 v1 视觉复刻基础上，进一步对齐用户提供的目标截图，纳入 HUD 面板、主题选择器、建筑形态差异、光缆粒子强化、性能档位等要素。
>
> **版本**：v2.0
> **最后更新**：2026-07-18
> **关联文档**：`.docs/UI_DESIGN_SYSTEM.md`、`.docs/VFX_DESIGN.md`、`2026-07-18-visual-replication-design.md`

## 1. 目标与范围

### 1.1 目标

让 Procession 主界面在静态帧和动态运行时都达到参考截图的完成度：

- 高密度的发光晶体建筑群，中心高、边缘低，冷暖对比强烈；
- 粗壮的弧形发光光缆与沿缆流动的数据粒子；
- 深空背景 + 远景城市剪影 + 星尘大气；
- 左上角 HUD 系统面板与右下角 2×2 主题选择器；
- 进程标签悬浮于建筑上方，颜色跟随建筑；
- 45° 电影感俯视相机与平滑的聚焦动画；
- 视觉优先，但提供高/中/低三档质量并支持 FPS 自适应降级。

### 1.2 范围

**包含：**

- 建筑形态与材质系统（晶体玻璃、窗户网格、能量核心、顶部结构差异）；
- 进程标签样式（颜色、发光、hover 状态）；
- 光缆几何、shader 发光、扫描线与流动粒子；
- 大气、星尘、远景背景城市；
- 地面平台与 subtle 网格；
- 相机默认角度、FOV、聚焦动画；
- HUD 面板（CPU / MEM / NET / TEMP / FPS）；
- 主题选择器（默认 / 深空 / 极光 / 熔岩）与缩略图；
- 性能档位与 FPS 自适应策略。

**不包含：**

- 全新的数据收集后端逻辑；
- 多窗口/设置页/插件市场等非主界面功能；
- 建筑内部结构或进程详情弹窗的重设计（沿用现有 `ProcessPopup`）。

### 1.3 设计原则

- **截图驱动**：每一项改动的验收标准是“和参考图叠在一起差距最小”。
- **视觉优先，性能兜底**：默认最高画质，但提供清晰降级路径。
- **数据即形态**：建筑高度、颜色、顶部结构必须能直观反映进程属性与负载。
- **组件隔离**：HUD、主题选择器、建筑、光缆、大气、相机各自独立，便于分别迭代与按档位开关。

## 2. 视觉氛围与色彩系统

### 2.1 背景与空间

- 背景色：`#03040a`（接近纯黑的深空）。
- 使用 `FogExp2`，密度 0.018，让远景自然隐入黑暗。
- 星尘粒子 800–1000 个，大小 0.05–0.15，极慢漂移并带闪烁。
- 远景城市分布在大半径圆环上，形成“城市只是更大数字世界一隅”的深度感。

### 2.2 色彩映射

| 用途 | 色值 | 说明 |
|------|------|------|
| 普通进程建筑 | `#4aa8ff` → `#00e5ff` | 底部冷蓝向顶部电光青渐变 |
| 高负载建筑 | `#ff9d3a` → `#ffb84d` | CPU/内存高的进程，琥珀色发光 |
| 数据库 / 网络服务 | `#9d7bff` | 紫色系，如 mysql、db server |
| 系统 / 浏览器 / 工具 | `#00e5ff` / `#5ce1a8` | 浏览器偏青，部分系统工具偏绿 |
| 脉冲高光 | `#ffffff` | 建筑顶部、选中轮廓、脉冲包 |
| 进程标签 | 与建筑 emissive 同色 | 带深色文字阴影，hover 变白放大 |
| 光缆 | 按协议分色 | tcp 蓝、udp 绿、http/https 青、dns 紫、ssh 绿 |
| 地面网格 | `#0a0a12` ~ `#10101a` | 极暗，只在建筑底部附近可见 |
| 背景远景建筑 | `#1a2a40` / `#1a1a2e` | 低亮度剪影，亮度 < 前景 20% |

### 2.3 后处理 / Bloom

- `strength`：0.45–0.6（增强发光晕影）。
- `threshold`：0.4–0.5（让更多自发光区域产生 Bloom）。
- `radius`：0.4–0.5（避免过度发糊）。
- 低档关闭 Bloom 以节省开销。

### 2.4 光照

- 环境光极弱：`ambientIntensity` ≈ 0.15。
- 主光源来自建筑自发光与能量流，避免额外方向光破坏夜景氛围。
- 高塔顶部可添加微弱垂直体积光柱（高档开启）。

## 3. 建筑形态与材质系统

### 3.1 建筑几何差异

基础形态仍为长方体，但按进程类型在顶部加入差异化结构：

| 进程类型 | 顶部结构 | 代表 |
|----------|----------|------|
| 浏览器 / 信息中心 | 双塔或分裂顶部 | chrome、edge |
| 编辑器 / 工业工具 | 平顶 + 顶部光带 | vs code、terminal |
| 数据库 / 网络服务 | 水晶棱柱尖顶 | mysql、db server |
| 系统 / 守护进程 | 坚固柱状 + 环状光环 | system、svchost |
| 普通应用 | 标准锥形或平顶 | wechat、dingtalk |

差异化仅通过 shader 或少量额外几何实现，不破坏 InstancedMesh 批量渲染。

### 3.2 建筑尺寸映射

- 底面：0.5–0.8（重要进程更粗）。
- 高度：由 CPU + 内存综合决定，高负载进程更高。
- 布局：中心区域集中高建筑，边缘填充矮建筑，整体呈放射/聚落分布。

### 3.3 材质 Shader

`BuildingCluster.tsx` 使用自定义 `ShaderMaterial`：

- **半透明晶体外壳**：alpha 0.85–0.95，Fresnel 边缘光。
- **内部能量核心**：建筑中心有亮度更高的发光柱。
- **垂直渐变**：底部暗色 → 顶部亮色，高负载建筑整体色相偏向琥珀。
- **窗户网格**：通过 fract UV 生成，夜间窗户随机点亮并缓慢闪烁。
- **顶部 cap glow**：建筑顶面强烈自发光。
- **能量扫描线**：沿建筑表面缓慢上移，扫描速度映射 CPU 占用。

### 3.4 建筑底部光环

- 每个建筑底部添加一个 `Ring` 或 `Torus`，颜色与建筑一致。
- 光环随建筑诞生动画从地面扩散升起，死亡时收缩消失。
- 高档显示完整光环，中低档可简化为贴地圆盘或关闭。

### 3.5 生命周期动画

- **诞生**：建筑从 y=0 升起至目标高度，scale 从 0→1，底部光环扩散，顶部发光逐渐点亮。
- **死亡**：scale 从 1→0，建筑下沉并伴随粒子消散，光环收缩。
- 保留现有的 `lifecycleRef` 状态机，但动画曲线改为 ease-out-back（诞生）和 ease-in（死亡）。

## 4. 进程标签

### 4.1 位置与样式

- 使用 `@react-three/drei` 的 `Html`，位置 `[x, height + 0.9, z]`。
- 字体大小 11–13px，字重 500，letter-spacing 0.02em。
- 文字颜色与建筑 emissive 同色，带 1px 深色文字阴影。
- 始终面向相机，透视缩放自然。

### 4.2 交互状态

- **默认**：与建筑同色，透明度 0.9。
- **hover**：颜色变白，scale 1.15，显示完整进程名与关键指标。
- **选中**：保持高亮，直到取消选择或选择其他建筑。
- 远景极小建筑（height < 1.0）可省略标签或降至 30% 透明度。

## 5. 光缆系统与数据流

### 5.1 光缆几何

- 使用 `TubeGeometry`，半径 0.14–0.18，分段 32。
- 路径为 `CatmullRomCurve3`，中间控制点抬高形成弧线。
- 两端有透明度衰减，避免生硬截断。

### 5.2 光缆 Shader

- 自定义 `ShaderMaterial`，`AdditiveBlending`，`depthWrite: false`。
- Fresnel 边缘光让光缆中心更亮。
- 沿 UV.x 移动的扫描线模拟数据包传输。
- 悬停建筑时，相关光缆加粗、变亮，非相关光缆变暗。

### 5.3 流动粒子

- 使用 `THREE.Points` + 径向发光 sprite（CanvasTexture 生成）。
- 粒子大小 0.3–0.5，颜色与光缆一致并混入 25% 白色。
- 速度按协议区分：tcp 慢、udp 快、http/https 中等、ssh 稳。
- 偶尔出现“脉冲包”：一个更亮、更快的粒子沿电缆闪过。
- 高档每根电缆 4–6 个粒子，中档 2–3 个，低档关闭。

## 6. 大气、背景与地面

### 6.1 星尘粒子

- 800–1000 个点，颜色偏冷青/白，大小 0.05–0.15。
- 运动极慢，带随机相位闪烁。
- 低档减少至 200 或关闭。

### 6.2 远景城市

- `CityBackground.tsx` 使用 `InstancedMesh`，数量 800–1200。
- 分布在大半径圆环（inner 28，outer 70）。
- 建筑矮小（0.5–4.5），颜色深蓝灰，仅顶部微光。
- 通过 LOD 控制：相机距离近时淡出或降低数量。

### 6.3 地面

- 圆形平台，颜色 `#0c0c14`，透明度 0.85。
- `GridHelper` 尺寸 120，分割 100，颜色 `#0a0a12` / `#03040a`，极其 subtle。
- 平台边缘自然融入背景，无硬边界。

## 7. 相机与构图

### 7.1 默认视角

- 位置 `[24, 30, 24]`，目标 `[0, 0, 0]`，FOV 40。
- 约 45° 俯视，强调城市布局和中心高建筑群。

### 7.2 控制限制

- `OrbitControls` 允许旋转、缩放，但限制 `minPolarAngle` 避免穿透地面。
- `maxDistance` 70，`minDistance` 8。

### 7.3 聚焦动画

- 双击建筑时，镜头平滑移动到建筑侧上方，目标点位于建筑顶部。
- 使用 `Vector3.lerp` 或 tween，时长 0.8s，ease-in-out。
- 聚焦过程中自动旋转暂停。

### 7.4 自动旋转

- 默认开启，速度 0.6。
- 演示/屏保模式下速度 1.2。
- 用户交互后 3 秒无操作再恢复旋转。

## 8. HUD 与主题选择器

### 8.1 HUD 面板

新建 `HudPanel.tsx`，固定在左上角：

| 指标 | 展示方式 | 视觉 |
|------|----------|------|
| CPU | 进度条 + 百分比 | 高负载时条色变琥珀 |
| MEM | 进度条 + 已用/总量 | 蓝色条 |
| NET | ↑ 上行 / ↓ 下行 | 青色/绿色数字 |
| TEMP | CPU / GPU 温度 | 温度高时变琥珀 |
| FPS | 数字 | 低于 30 变红 |

- 面板背景：半透明深色（rgba(3,4,10,0.65)），圆角 12px，边框 1px `rgba(255,255,255,0.08)`。
- 字体使用等宽或系统无衬线，字号 12–14px。

### 8.2 主题选择器

新建 `ThemeSelector.tsx`，固定在右下角：

- 2×2 卡片布局，每张卡片带主题缩略图 + 中文名称。
- 主题：默认主题、深空主题、极光主题、熔岩主题。
- 当前选中主题有外发光边框。
- 切换主题时，整个场景颜色、Bloom、背景平滑过渡。

### 8.3 底部 Slogan

- 文字：`Procession · 进程列队，系统成诗`。
- 位置：底部居中，距底 26px。
- 样式：13px，letter-spacing 0.12em，透明度 0.75，带深色阴影。

## 9. 性能与质量档位

### 9.1 档位定义

| 档位 | 建筑 | 背景城市 | 粒子 | Bloom | 光环/光柱 | 触发条件 |
|------|------|----------|------|-------|-----------|----------|
| 高 | 200 | 1000 | 完整 | 开 | 开 | 默认；FPS > 50 |
| 中 | 120 | 400 | 减半 | 简化 | 简化 | FPS 30–50 或用户手动 |
| 低 | 60 | 0 | 关闭 | 关 | 关 | FPS < 30 或用户手动 |

### 9.2 自适应策略

- 每 1 秒采样一次 FPS。
- 连续 3 秒低于阈值则降档；连续 5 秒高于阈值则升档。
- 用户可手动锁定档位，手动优先级高于自动。
- 档位通过 React Context（`QualityContext`）下发，所有视觉组件读取并条件渲染。

### 9.3 组件级优化

- `BuildingCluster`：InstancedMesh，`frustumCulled`。
- `CityBackground`：InstancedMesh + LOD。
- `CableSystem`：合并 geometry，单一 draw call。
- `CableFlow`：按档位控制粒子数量，低档 `return null`。
- 建筑底部光环：高档 InstancedMesh，中低档可关闭。

## 10. 数据映射

### 10.1 建筑到进程

- 颜色：`colorForProcess(process, theme)`，综合进程名、CPU、类型。
- 高度：`height = base + cpuFactor + memFactor`，并做非线性压缩避免极端值。
- 形态顶部：根据 `process.name` 关键词（chrome/vscode/mysql/system 等）决定。
- 标签：仅显示重要进程，或全部显示但远景降透明。

### 10.2 光缆到连接

- 起点：建筑出口（高度 0.7 处）。
- 终点：根据 `remote_addr` 哈希映射到城市外围的固定位置。
- 颜色：按 `protocol` 映射。
- 粒子速度：按 `protocol` 映射。

### 10.3 HUD 数据

- 绑定 `displaySnapshot`：CPU、MEM、NET、TEMP、FPS。
- 历史回放模式下 HUD 显示当前回放帧数据，并添加“非实时”提示。

## 11. 测试与验收

### 11.1 自动化测试

- `npm test -- --run` 保持通过。
- `npx tsc --noEmit` 无类型错误。
- `npm run build` 构建成功。
- `cargo clippy` 无警告。

### 11.2 视觉验收

- 运行应用后截取全屏，与参考图叠放对比。
- 检查项：
  - 建筑密度与中心高边缘低布局；
  - 高负载建筑琥珀色是否突出；
  - 光缆粗细、颜色、粒子流动；
  - HUD 位置、内容、颜色；
  - 主题选择器位置与样式；
  - 底部 slogan 位置与样式；
  - 相机俯视角度；
  - 远景背景城市深度。

### 11.3 性能验收

- 高档在 M 系列 Mac / 中高端 PC 上目标 60 FPS。
- 中档目标 45 FPS 以上。
- 低档目标 30 FPS 以上。
